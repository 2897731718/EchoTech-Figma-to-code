## 组件识别决策树

```
                              组件标签
                                 │
                 ┌───────────────┴───────────────┐
                 ↓                               ↓
        命中 aliases？                      未命中 aliases
           (index.json)
                 │                               │
                 ↓                               ↓
        ┌───────────────┐              命中业务组件映射表？
        │ 💙 分子组件    │               (figma-context.md)
        │ NavigationBar │                        │
        │ → DuNavBar    │              ┌─────────┴─────────┐
        └───────┬───────┘              ↓                   ↓
                │                     命中                未命中
                ↓                      │                   │
        属性参数匹配？                  ↓                   ↓
         ┌──────┴──────┐        ┌───────────┐      ┌─────────────┐
         ↓             ↓        │ 👻 业务组件 │      │ ⚠️ 未识别    │
       完全匹配     部分匹配     │ 直接使用   │      │ 待递归生成  │
         │             │        │ 属性调用   │      │ 或更新组件库│
         ↓             ↓        └───────────┘      └─────────────┘
     正常翻译     ⚠️ 告警：
                 属性不完全匹配
```

## 循环点汇总

| # | 位置 | 触发条件 | 动作 |
|---|------|----------|------|
| ① | 第五步 | 还有未处理的组件 | 询问用户 → 递归生成 → 处理下一个 |

---

## 告警类型

| 告警 | 触发条件 | 输出示例 |
|------|----------|----------|
| ⚠️ 属性参数不完全匹配 | 💙 分子组件匹配，但骨架属性与规则不完全对应 | `terminal` 未定义，建议用 `extClass` 透传 |
| ⚠️ 未识别的组件 | INSTANCE 未命中 aliases 也未命中业务映射表 | 可能需要更新组件库或第五步递归生成 |

---

## 终止点

1. **骨架生成失败** → 输出错误信息 → 结束
2. **figma-context.md 不存在** → 提示 init → 结束
3. **所有组件处理完毕** → 输出最终代码 → 结束
